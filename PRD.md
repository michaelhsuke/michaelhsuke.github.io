# 业娱家 (webook.pub) - 产品需求文档

> 分享有趣的生活内容，顺便种草好物

## 1. 项目概述

### 1.1 项目定位

**业娱家 (webook.pub)** 是一个以「分享有趣内容」为核心的生活娱乐类内容网站。通过高质量的文章、播客推荐、短视频精选吸引泛人群用户，自然融入 CPS（Cost Per Sale）推广内容，实现内容变现。

### 1.2 核心信息

| 项目 | 内容 |
|------|------|
| **网站名称** | 业娱家 |
| **域名** | webook.pub |
| **Slogan** | 待定 |
| **设计风格** | 简约 |
| **目标用户** | 泛人群 |

### 1.3 核心价值

- **对读者**：一站式发现有趣、有用的生活娱乐内容
- **对运营者**：通过内容种草实现 CPS 推广变现
- **对广告主**：通过软性内容获取精准转化

## 3. 内容规划

### 3.1 内容类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **文章** | 深度图文内容，SEO 友好 | 好物测评、生活技巧、使用心得、盘点推荐 |
| **播客** | 播客推荐与收听引导（外链或内嵌播放器） | 小众播客推荐、单集精选、播客主题合集 |
| **短视频** | 短视频精选与推荐（外链或内嵌） | TikTok/抖音精选、YouTube Shorts、B站精选 |

### 3.2 内容分类（建议）

```
├── 好物推荐（侧重 CPS 转化）
├── 生活灵感（日常、旅行、美食）
├── 播客精选（音频内容推荐）
├── 短视频精选（视频内容推荐）
├── 工具 & 效率（软件/工具推广）
└── 闲聊 & 观点（软性引流/人设打造）
```

### 3.3 CPS 推广策略

推广内容以「原生内容」形式融入，不单独设推广专区，保持阅读体验：

| 推广类型 | 形式 | 示例 |
|----------|------|------|
| **电商返利** | 好物推荐文中嵌入商品链接（淘宝客/京东联盟） | "这 5 件厨房好物让我幸福感拉满" |
| **软件/工具** | 效率工具推荐文中嵌入注册/下载链接 | "这 3 个 App 改变了我的工作方式" |
| **平台引流** | 文章末尾/侧边栏引导关注公众号等 | 公众号二维码、微信号 |

**原则**：先有内容价值，再自然植入推广；推广内容占比不超过总内容的 30%。

## 4. 技术方案

### 4.1 方案对比与选型

考虑到运营者是 Vue3 前端开发工程师，以下三个方案的对比：

| 维度 | 方案 A: VitePress | 方案 B: Vite + Vue3 全自定义 | 方案 C: Astro |
|------|-------------------|------------------------------|---------------|
| **开发效率** | 高（开箱即用） | 低（需自建路由/SSG/主题） | 中等 |
| **定制自由度** | 中（主题扩展） | 高（完全自由） | 高 |
| **Vue3 组件复用** | 支持（可在主题中用 Vue3 组件） | 完全支持 | 支持（需 React/Vue islands） |
| **Markdown 支持** | 内置且强大 | 需手动集成 markdown-it 等 | 内置（MDX） |
| **SEO** | 内置 SSG，天然 SEO 友好 | 需自行配置 SSG | 内置 SSG |
| **学习成本** | 低（熟悉 Vite + Vue3 即可） | 低（纯 Vue3 生态） | 中（需学 Astro） |
| **社区/生态** | 活跃，主题/插件丰富 | 依赖 Vue3 生态 | 活跃，偏多框架 |
| **性能** | 极好（纯静态） | 取决于实现 | 极好（部分水合） |

### 4.2 推荐方案：VitePress + 深度主题定制

**推荐理由**：

1. **技术栈契合**：VitePress 底层就是 Vite + Vue3，你的 Vue3 经验可以直接用在主题定制上
2. **开箱即用**：SSG、Markdown 渲染、路由、搜索、SEO 等核心功能无需自建
3. **定制能力足够**：通过 `theme/index.ts` + Vue3 组件可以完全自定义布局和样式
4. **内容优先**：Markdown 驱动的内容站点，VitePress 是最佳选择
5. **部署简单**：原生支持 GitHub Pages，配置最少

**对于需要深度定制的场景**，可以在 VitePress 主题中使用自定义 Vue3 组件（如 CPS 卡片组件、视频嵌入组件、播客播放器组件等），完美结合开箱即用和定制自由度。

### 4.3 技术栈明细

| 项目 | 选型 | 说明 |
|------|------|------|
| **框架** | VitePress 1.x | 基于 Vite + Vue3 的静态站点生成器 |
| **样式** | UnoCSS | 原子化 CSS，配合 VitePress 使用体验好，Vue3 生态友好 |
| **访问统计** | 不蒜子 + 百度统计 | 不蒜子做简单的 PV/UV 展示；百度统计做详细数据分析 |
| **评论** | Giscus | 基于 GitHub Discussions，免费、无后端、国内可访问 |
| **搜索** | VitePress 内置本地搜索 | 无需第三方服务，支持中文分词 |
| **RSS** | vitepress-plugin-rss | 自动生成 RSS feed |
| **部署** | GitHub Pages + GitHub Actions | 免费，支持自定义域名，自动部署 |
| **域名** | webook.pub | 通过 CNAME 解析到 GitHub Pages |

### 4.4 自定义 Vue3 组件规划

在 VitePress 主题中注册以下自定义组件，直接在 Markdown 中使用：

```
├── CpsCard.vue          # CPS 推广卡片（商品图+名称+价格+跳转按钮）
├── VideoEmbed.vue       # 视频嵌入组件（B站/YouTube/抖音）
├── PodcastCard.vue      # 播客推荐卡片（封面+标题+平台+链接）
├── ProductGrid.vue      # 商品网格展示（多商品对比推荐）
├── AffiliateBadge.vue   # 推广标识徽章（"推广" / "广告" 标注）
└── BuJuan.vue           # 不蒜子 PV/UV 组件
```

Markdown 中使用示例：

```md
<!-- CPS 推广卡片 -->
<CpsCard
  title="戴森 V15 吸尘器"
  image="/images/dyson-v15.jpg"
  price="3990"
  platform="jd"
  link="https://union.jd.com/..."
/>

<!-- 视频嵌入 -->
<VideoEmbed platform="bilibili" bvid="BV1xx411c7mD" />

<!-- 播客卡片 -->
<PodcastCard
  title="随机波动"
  cover="/images/podcast-cover.jpg"
  platform="xiaoyuzhou"
  link="https://www.xiaoyuzhoufm.com/..."
/>
```

### 4.6 部署流程

1. 推送代码到 GitHub 仓库的 `main` 分支
2. GitHub Actions 自动构建 VitePress 静态文件
3. 部署到 `gh-pages` 分支
4. GitHub Pages 从 `gh-pages` 分支提供服务
5. 域名 webook.pub 通过 CNAME 指向 `<username>.github.io`

## 5. 页面设计（简约风格）

### 5.1 设计原则

- **留白**：充足的间距，不拥挤
- **层次**：通过字号、颜色权重区分信息层级
- **克制**：不堆砌元素，每屏只传达一个核心信息
- **统一**：一致的间距系统、颜色系统、字体系统

### 5.2 首页

- **Hero 区域**：网站名「业娱家」+ Slogan + 简约背景
- **最新推荐**：最近发布的文章（卡片式布局，简洁信息：标题 + 日期 + 摘要）
- **分类导航**：图标 + 文字的分类快捷入口
- **精选推荐**：编辑精选的高质量内容

### 5.3 文章/内容详情页

- 标题 + 发布日期 + 分类标签 + 预估阅读时长
- 正文内容（Markdown 渲染，排版舒适）
- CPS 推广卡片（自然嵌入文中或文末）
- 相关推荐（底部，同分类文章）
- 版权声明 / 推广声明

### 5.4 分类列表页

- 分类标题 + 一句话描述
- 内容列表（卡片式，封面图 + 标题 + 日期）

### 5.5 关于页

- 网站介绍
- 运营者简介（人设打造）
- 联系方式
- 免责声明

## 7. CPS 链接管理

### 7.1 链接处理方案

**MVP 阶段：直接嵌入**
- 直接在 Markdown 文中放置 CPS 链接，通过自定义组件渲染
- 配合 frontmatter 标记管理推广内容

**后续优化：统一短链跳转**
- 自建短链服务（如 Shlink / Yourls）
- 所有 CPS 链接通过短链跳转，便于统计和统一管理

### 7.2 推广标记规范

在内容 frontmatter 中标记推广信息：

```yaml
---
title: 这 5 件厨房好物让我幸福感拉满
date: 2026-03-27
category: 好物推荐
tags: [厨房, 好物, 生活]
cover: /images/kitchen-goods.jpg
cps: true
cps_products:
  - name: 商品A
    platform: jd
    link: "https://union.jd.com/..."
  - name: 商品B
    platform: taobao
    link: "https://s.click.taobao.com/..."
---
```

## 10. 风险与注意事项

- **内容质量**：避免纯广告水文，保持内容价值是核心
- **合规性**：推广内容需标注"推广"或"广告"标识，遵守《广告法》
- **隐私合规**：使用统计/评论功能需配置隐私政策页面
- **GitHub Pages 限制**：仓库大小建议 < 1GB，单次构建 < 10 分钟
- **CPS 链接稳定性**：第三方推广链接可能失效，需定期检查
